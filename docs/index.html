DEMON

Diffusion Engine for Musical Orchestrated Noise

A real-time diffusion engine for interactive music generation. Twist a knob, hear the change in the next 102 milliseconds.

102ms per tick 12.3 gen/s 25Hz per-frame control consumer GPU
GitHub PyPI Paper

What it is

DEMON wraps the ACE-Step v1.5 model with a streaming pipeline that produces one finished generation every ~80 milliseconds on an RTX 5090, and exposes denoising as a per-frame control surface at 25Hz. Six per-frame curves (velocity, SDE source blending, guidance, noise injection, init mixing, latent morphing) replace the usual scalar parameters, so you can shape an entire generation continuously instead of picking one number.

The result is something close to a synthesizer for diffusion: a MIDI knob, webcam motion, or any 25Hz signal can drive the generation while it plays. Same model, same weights as ACE-Step. New engine around them.

Quick start

Requires Python 3.11 and a CUDA GPU. ACE-Step weights download automatically on first run.

# clone and install $ git clone https://github.com/RyanOnTheInside/DEMON $ cd DEMON $ uv sync # launch the realtime MIDI demo $ uv run python -m demos.realtime_motion_graph --midi --audio source.wav

Full setup, TensorRT engine builds, and the Session API are documented in the repo README.
